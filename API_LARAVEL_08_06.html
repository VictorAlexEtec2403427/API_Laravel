📡 Criando uma API simples com Laravel + SQLite

✏️ Material didático para Ensino Médio e Técnico — Testando com Postman (sem autenticação)
💡 Objetivo da aula: Entender o que é uma API, construir uma API RESTful completa (CRUD) usando Laravel, banco SQLite e testar todos os endpoints no Postman.
✅ Pré-requisitos: PHP (>=8.1), Composer, SQLite3 instalado, Postman instalado (ou extensão navegador).

🔰 1. O que é uma API? Por que usar Laravel e SQLite?

Uma API (Application Programming Interface) permite que sistemas conversem entre si. No contexto web, uma API RESTful recebe requisições HTTP (GET, POST, PUT, DELETE) e devolve respostas em JSON.
O Laravel é um framework PHP poderoso e elegante, que facilita criação de rotas, controllers e migrations. O SQLite é um banco de dados relacional leve, excelente para estudos e projetos pequenos — tudo armazenado em um único arquivo.

🛠️ 2. Criando o projeto Laravel e configurando SQLite

2.1 Criar novo projeto (via Composer)

composer create-project laravel/laravel minha-api-laravel
cd minha-api-laravel

Após criar, acesse a pasta.

2.2 Configurar SQLite

🔹 No arquivo .env (raiz do projeto), altere as linhas de banco de dados:

DB_CONNECTION=sqlite
# DB_HOST, DB_PORT, DB_DATABASE, DB_USERNAME, DB_PASSWORD podem ser comentados ou removidos

# Certifique-se que não há linhas conflitando com mysql/pgsql.

🔹 Crie um arquivo vazio: database/database.sqlite (Linux/Mac: touch database/database.sqlite ; Windows: basta criar um arquivo em branco com esse nome).
🔹 Pronto! O Laravel usará esse arquivo como banco.

🗄️ 3. Criando Migration e Model (Tabela "produtos")

Vamos construir uma API de gerenciamento de produtos (apenas para exemplo).

3.1 Gerar migration e model

php artisan make:model Produto -m

O comando acima cria o model app/Models/Produto.php e uma migration em database/migrations/.

3.2 Editar migration (definir colunas)

Abra o arquivo migration gerado (ex: 2025_xx_xx_xxxxxx_create_produtos_table.php) e edite o método up():

public function up(): void
{
Schema::create('produtos', function (Blueprint $table) {
$table->id();
$table->string('nome', 100);
$table->text('descricao')->nullable();
$table->decimal('preco', 8, 2);
$table->integer('quantidade')->default(0);
$table->timestamps();
});
}

3.3 Executar migração (criar tabela)

php artisan migrate

Verifique se não há erros. A tabela produtos foi criada no SQLite!

🌐 4. Criando o Controller da API (sem autenticação)

Vamos gerar um controller específico para API RESTful:

php artisan make:controller Api/ProdutoController --api

O arquivo será criado em app/Http/Controllers/Api/ProdutoController.php.

Edite o conteúdo com os métodos básicos: index (listar), store (criar), show (buscar um), update (atualizar) e destroy (deletar). Código completo:

<?php

namespace App\Http\Controllers\Api;

use App\Models\Produto;
use Illuminate\Http\Request;
use App\Http\Controllers\Controller;

class ProdutoController extends Controller
{
// Listar todos os produtos
public function index()
{
$produtos = Produto::all();
return response()->json($produtos);
}

// Criar novo produto
public function store(Request $request)
{
$request->validate([
'nome' => 'required|string|max:100',
'preco' => 'required|numeric|min:0',
'quantidade' => 'integer|min:0',
'descricao' => 'nullable|string'
]);

$produto = Produto::create($request->all());
return response()->json($produto, 201);
}

// Exibir um produto específico
public function show($id)
{
$produto = Produto::find($id);
if(!$produto) {
return response()->json(['message' => 'Produto não encontrado'], 404);
}
return response()->json($produto);
}

// Atualizar produto
public function update(Request $request, $id)
{
$produto = Produto::find($id);
if(!$produto) {
return response()->json(['message' => 'Produto não encontrado'], 404);
}

$request->validate([
'nome' => 'sometimes|string|max:100',
'preco' => 'sometimes|numeric|min:0',
'quantidade' => 'sometimes|integer|min:0',
'descricao' => 'nullable|string'
]);

$produto->update($request->all());
return response()->json($produto);
}

// Deletar produto
public function destroy($id)
{
$produto = Produto::find($id);
if(!$produto) {
return response()->json(['message' => 'Produto não encontrado'], 404);
}
$produto->delete();
return response()->json(['message' => 'Produto removido com sucesso']);
}
}

🧭 5. Definindo as Rotas da API (routes/api.php)

Abra o arquivo routes/api.php e registre as rotas RESTful:

<?php

use Illuminate\Support\Facades\Route;
use App\Http\Controllers\Api\ProdutoController;

Route::apiResource('produtos', ProdutoController::class);

Pronto! Rotas geradas automaticamente:

  • GET /api/produtos → index
  • POST /api/produtos → store
  • GET /api/produtos/{id} → show
  • PUT/PATCH /api/produtos/{id} → update
  • DELETE /api/produtos/{id} → destroy

🚀 6. Servindo a aplicação Laravel

No terminal, dentro da pasta do projeto, execute:

php artisan serve

O servidor de desenvolvimento será iniciado em http://127.0.0.1:8000. Nossa API estará disponível em http://127.0.0.1:8000/api/produtos

⚠️ Lembre-se: mantenha o terminal rodando. Para testar no Postman, a API estará ativa localmente.

🧪 7. Testando a API no Postman (passo a passo)

Abra o Postman (ou o cliente REST de sua preferência). Vamos realizar todos os testes sem autenticação.

🔹 7.1 Criar um novo produto (POST)

  • Método: POST
  • URL: http://127.0.0.1:8000/api/produtos
  • Headers: Content-Type: application/json
  • Body (JSON):
{
"nome": "Teclado Mecânico",
"descricao": "Teclado RGB switch marrom",
"preco": 199.90,
"quantidade": 15
}

✅ Resposta esperada: 201 Created + dados do produto com ID gerado.

🔹 7.2 Listar todos os produtos (GET)

  • Método: GET
  • URL: http://127.0.0.1:8000/api/produtos

✅ Retorna array de produtos cadastrados (JSON).

🔹 7.3 Buscar um produto por ID (GET)

  • Método: GET
  • URL: http://127.0.0.1:8000/api/produtos/1

✅ Exibe detalhes do produto com id=1. Caso não exista, retorna 404 com mensagem.

🔹 7.4 Atualizar um produto (PUT)

  • Método: PUT (ou PATCH)
  • URL: http://127.0.0.1:8000/api/produtos/1
  • Body JSON (exemplo):
{
"preco": 179.90,
"quantidade": 22
}

✅ Resposta com os dados atualizados.

🔹 7.5 Deletar um produto (DELETE)

  • Método: DELETE
  • URL: http://127.0.0.1:8000/api/produtos/1

✅ Retorna mensagem de sucesso: {"message": "Produto removido com sucesso"}.

📌 Dica para testar: após deletar, tente buscar o mesmo ID e receberá 404. Isso prova que o CRUD está funcionando perfeitamente.

📊 8. Exemplo de fluxo completo + prints mentais

Imagine que você cadastrou três produtos com POST, depois chamou o GET /api/produtos e recebeu algo como:

[
{"id":1,"nome":"Mouse Gamer","preco":89.90,"quantidade":10},
{"id":2,"nome":"Monitor LED","preco":759.00,"quantidade":3}
]

E então fez um PUT no produto 2 alterando o preço. Simples, eficiente e sem necessidade de tokens ou autenticação (perfeito para aprendizado técnico).

🔧 10. Possíveis problemas e soluções

  • SQLite não encontrado: certifique-se que a extensão pdo_sqlite está habilitada no php.ini.
  • Erro 404 no Postman: confirme se o servidor está rodando (php artisan serve) e a URL está correta.
  • Validação falhando: verifique os campos obrigatórios como "nome" e "preco" no JSON.
  • Migration não executa: delete o arquivo database.sqlite (caso corrompido) e recrie.
